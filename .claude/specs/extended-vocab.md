# Goal
- for each `tense` provide a new vocabulary.
- for each `example` provide a new flashcard with sentence at front and meaning at back.

# Implementation detail request
- each vocabulary data now should have:
    1. spelling (kanji or just spelling)
    2. meaning
    3. type by the enum as explain below.
- The data is provided in markdown following the below pattern:

```
### [spelling]
kanji: [kanji]
meaning: to leave; to exit; to go out、to start
type: verb (dictionary)
tense: present : [spelling] 
tense: past-formal : [spelling]
tense: past-informal (ta-form): [spelling]
tense: negative-present-formal:
tense: negative-present-informal (nai-form): [spelling]
tense: negative-past-formal  : [spelling]
tense: negative-past-informal (nai-form) : [spelling]
tense: potential-form-formal : [spelling]
tense: potential-form-informal : [spelling]
tense: potential-negative-form : [spelling]
tense: potential-negative-form-informal (nai-form) : [spelling]
tense: te-form : [spelling]
example: [japanese sentence] : [sentence-meaning]
```
* the header becomes a word with "meaning" and "type" such as "verb (dictionary)"
* subsequence `tense` such as `tense : present : でます` can define a new word with:
  * spelling is provided after the `:` such as でます
  * same meaning
  * `type` is `verb (present)`
* any `tense` that doesn't have the spelling will not provide a new word. Or if `tense` is not provided, then there is no new vocab

- other files also have same patterns 
- should define enum where the enum is the type such as:
  ```
  PAST_FORMAL
  PAST_INFORMAL
  POTENTIAL_FORM_FORMAL
  ```
  Note that the type in parenthesis `()` is not needed to be included in enum's name.

- Vocabulary Flashcard only needs spelling/kanji at front and meaning in back, same as before. 
- Sentence Flashcard has the sentence provided by `[japanese sentence]` and `[sentence meaning]`.
- Sentence data should be in a separate json file, generated by "Generate Exercise" button.
